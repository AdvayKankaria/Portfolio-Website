**Theme:** Graphite Mono + Red — precision engineering meets controlled intensity
**Default mode:** Dark (the designed experience). Light is a functional fallback only.

---

## Concept

Graphite Mono + Red = a dark engineering portfolio with one precise strike of red.

- **Graphite** — cool machined steel / carbon fiber. Metallic gray with a subtle cool-blue
  undertone (OKLCH hue ≈ 260). Never warm gray (hue > 50 is forbidden).
- **Mono** — pure black/white editorial contrast pushed to its limit. Typography is the design.
- **Red** — the only chromatic signal. A scalpel, not decoration. Appears **only** where a user
  is about to interact, is interacting, or should be directed to interact.

**Where red is PERMITTED (exactly these):** primary CTA fill, ghost-button border/text,
active nav indicator line, custom cursor ring, scroll-progress bar, project-card hover
(2px left border + title → red-400), current-section number, loading-screen blink cursor.

**Where red is FORBIDDEN:** backgrounds/surfaces, cards/panels, body copy, section headings,
skill/stack badges, resting-state icons, any non-interactive decoration. Max 3 reds per viewport.

### Typography

- **Display + Headings:** `Geist Mono` (var `--font-mono`) — weight 500, letter-spacing −0.02em
  (−0.03em on display). Line-height 1.1. Used for h1/h2/h3, navbar logo, section numbers, stat
  counters, code, badges, buttons.
- **Body + UI:** `Geist` (var `--font-sans`) — weight 400, 500 for labels. High legibility.
- **Buttons:** font-mono, UPPERCASE, `tracking-widest`.
- **No italics** (italic reads warm; this theme is cold). No third font. No emoji.
- Loaded via `next/font/google` (`Geist`, `Geist_Mono`). Zero FOUT.

### Border / Radius / Surface

- **Radius:** `--radius: 0.25rem`. Sharp corners everywhere **except badges** (pill / full).
  Card corners must never exceed 0.25rem.
- **Surfaces:** only `--background`, `--card`, `--muted`. **No gradients on surfaces.**
- **Cards use border, not shadow.** 1px solid `--border`.

### Shadows / Glows (only TWO glows allowed)

| Token | Value | Apply to |
|-------|-------|----------|
| Graphite glow (ambient) | `0 0 40px -10px oklch(0.55 0.01 260 / 0.4)` | hero 3D canvas only |
| Red glow (active) | `0 0 30px -8px oklch(0.58 0.22 27 / 0.5)` | primary CTA on hover only |

No other box-shadows. Cards never use shadow.

### Animation Tokens

| Token | Value |
|-------|-------|
| `--duration-instant` | 100ms |
| `--duration-fast` | 200ms |
| `--duration-base` | 350ms |
| `--duration-slow` | 600ms |
| `--duration-page` | 900ms |
| `--ease-out-expo` | `cubic-bezier(0.16, 1, 0.3, 1)` |
| `--ease-in-out-expo` | `cubic-bezier(0.87, 0, 0.13, 1)` |

Every animation must serve **reveal**, **hierarchy**, or **feedback** — else remove it.
All motion respects `prefers-reduced-motion` (opacity-only fallback, no transforms).

## Component Specs

### Buttons — exactly three CVA variants, no fourth

**PRIMARY (red fill)** — one per section max, never two side by side.
- bg `--red-500`, text `oklch(0.98 0.005 260)`, font-mono uppercase tracking-widest, no border,
  radius 0.25rem, padding 12px 28px. Hover → `--red-400` + red glow. Active → `--red-600`.
  Framer: `whileHover scale 1.02`, `whileTap 0.98`. e.g. VIEW WORK, CONTACT ME, DOWNLOAD CV.

**GHOST (red border)** — secondary CTAs.
- transparent bg, 1px solid `--red-500`, text `--red-400`, font-mono uppercase tracking-widest.
  Hover → fill `oklch(0.58 0.22 27 / 0.08)` + border brightens to `--red-400`.
  Framer: `whileHover 1.01`, `whileTap 0.98`.

**GHOST STEEL (graphite border, no red)** — low-priority ("View Source", "Read More").
- transparent bg, 1px solid `--border`, text `--muted-foreground`, font-mono uppercase.
  Hover → border `--graphite-400`, text `--foreground`. No red.

### Cards

- bg `--card`, 1px solid `--border`, radius 0.25rem. **No shadow. No hover scale.**
- Hover state: `border-left: 2px solid var(--red-500)` slides in from width 0 over 200ms;
  bg shifts to `oklch(0.14 0.009 260)` over 200ms; title text → `--red-400` over 200ms.

### Inputs

- bg `--input`, 1px solid `--border`, radius 0.25rem, font-size 16px (never below).
- Focus: border → `--ring` (red), visible focus ring, no raw `outline:none` without replacement.
- `<label>` elements always — never placeholder-as-label.

### Section heading pattern

```
<span class="font-mono text-sm text-muted-foreground">01</span>   ← red when section in view
<h2 class="font-mono">About</h2>
```
The current in-view section's number turns `--red-500` via IntersectionObserver; others stay muted.

## Style Guidelines

**Style:** Dark engineering / terminal. High contrast. Graphite surfaces, one red strike.
**Key effects:** red left-border card reveal, red active-section number, magnetic CTA,
character-stagger text reveal, mouse-parallax 3D, scroll-draw timeline. Two glows only.

### Page Pattern — Section Order

1. Hero (name/role) → 2. About → 3. Skills → 4. Experience → 5. Projects → 6. Research →
7. Achievements → 8. GitHub → 9. Contact → 10. Footer.

---

## Anti-Patterns (Do NOT Use)

- ❌ Light mode as the primary experience (dark is designed; light is fallback)
- ❌ Warm grays (OKLCH hue > 50) — reads as a generic dark theme
- ❌ Any color besides red + graphite (no blue/purple/teal/green/orange)
- ❌ Red on surfaces, backgrounds, or headings; two red primaries in one viewport; red > 3× / viewport
- ❌ Gradient backgrounds on sections
- ❌ Card corners > 0.25rem; box-shadows instead of borders (except the two approved glows)
- ❌ Hover **scale** on cards (use the red left-border transition instead)
- ❌ Personality fonts, italics, emojis, centered body text
- ❌ Colored skill badges (graphite border + muted text only)
- ❌ Progress bars / percentages in Skills
- ❌ Emojis as icons — use Lucide / SVG
